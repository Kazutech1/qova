# Qova Circle Management: Technical & Functional Guide

This document outlines the architecture, features, and required API integrations for the newly implemented **Circle Detail** management module.

---

## 1. Module Overview
The Circle module is designed as a dynamic, nested navigation system within the Qova app. It utilizes `expo-router`'s dynamic segments (`[id]`) to provide unique management interfaces for individual savings circles (ROSCAs).

**Location:** `app/(circle)/[id]/`
**Theme:** Neon Industrial (utilizing `src/theme.ts`)

---

## 2. Navigation Architecture
The module uses a **Nested Tabs** layout with a custom header and a floating, glassmorphic bottom tab bar.

### Tabs Included:
- **OVERVIEW**: Performance hub and contribution management.
- **MEMBERS**: Participant list and turn-order tracking.
- **CHAT**: Real-time group communication.
- **HISTORY**: Transaction audit trail.
- **ADMIN**: Configuration console (for circle creators).

**Scroll-to-Hide:** The bottom tab bar automatically hides on scroll-down and reappears on scroll-up across all screens.

## 3. Screen-by-Screen Functionality

### A. Circle Overview (`index.tsx`)
- **Cycle Gauge**: Semi-circle visualizer tracking the current round (e.g., Round 4 of 10).
- **Contribution Pots**: Visual representation of "Current Round" vs "Overflow/Buffer" payments.
- **Definitive Paid State**: The UI transforms to a "Dark Mode" celebratory state with a **"ROUND CLEARED"** banner when the current round is fully paid.
- **Sticky Contribution Bar**: A fixed bar above the tabs for instant access to the payment flow.
- **Deposit Flow**: Triggered via the sticky bar, opens a modal with Bank Details (Account Number, Bank Name) and a manual "I have sent the money" confirmation.
- **Success Sequence**: A dedicated success state with reliability point rewards.

### B. Members (`members.tsx`)
- **Participant List**: Displays all members with their turn number.
- **Payment Verification**: Clear badges for **PAID** vs **PENDING** for the active round.
- **Avatar System**: Procedurally colored initials for easy member identification.

### C. Admin Console (`admin.tsx`)
- **Logic Toggles**: 
  - **Auto-Debit**: Enable/disable automated collections.
  - **Smart Reminders**: Automated chat pings for upcoming dues.
- **Admin Actions**: 
  - Edit Turn Order, Late Penalties, Modify Circle Rules, and Dissolve Circle.

## 4. Required API Endpoints

To move from mock data to production, the following endpoints are required:

### Circle Metadata
- `GET /v5/circles/{id}`
  - Returns: `pot_amount`, `current_round`, `total_rounds`, `contribution_amount`, `member_count`, `admin_id`.

### Personal Status
- `GET /v5/circles/{id}/me`
  - Returns: `is_paid`, `overflow_amount`, `total_earned`, `reliability_score`, `next_payout_date`.

### Members & Payments
- `GET /v5/circles/{id}/members`
  - Returns: Array of `{ name, turn_order, has_paid_current, status }`.
- `POST /v5/circles/{id}/contribute`
  - Payload: `{ amount, transaction_reference }`.

### Admin Operations
- `PATCH /v5/circles/{id}/config`
  - Payload: `{ auto_debit_enabled, reminders_enabled, rules: {...} }`.
- `POST /v5/circles/{id}/dissolve`
  - Action: Terminates the circle.
